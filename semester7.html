Teknik Kompilasi

  • Kegunaan Kompiler

    Kompiler adalah program yang mengubah kode sumber dari bahasa pemrograman tingkat tinggi ke bahasa mesin atau bahasa lain yang setara yang dapat dieksekusi oleh komputer. Kegunaan utama dari kompiler adalah:
    Penerjemahan Kode
    Optimasi Kode
    Deteksi Kesalahan
    Menghasilkan Kode Eksekusi
    Portabilitas

Struktur Kompiler

Compiler

Compiler biasanya terdiri dari beberapa komponen utama yang bekerja sama untuk menerjemahkan dan mengoptimalkan kode sumber. Struktur kompiler dibagi menjadi dua bagian utama: Front-End dan Back-End.

Analisis Awal

1. Lexical Analysis (Analisis Leksikal)
Mengubah aliran karakter dari kode sumber menjadi token (unit dasar seperti kata kunci, variabel, operator).
Alat yang digunakan: Lex.
2. Syntax Analysis (Analisis Sintaksis)
Menguraikan token menjadi struktur sintaksis, seperti pohon parse, untuk memastikan tata bahasa yang benar.
Alat yang digunakan: Yacc.
3. Semantic Analysis (Analisis Semantik)
Memeriksa semantik (arti) kode untuk memastikan tipe data yang benar dan operasi yang valid.
Melibatkan tabel simbol yang menyimpan informasi tentang variabel, fungsi, dll.

Optimasi

1. Intermediate Code Generation (Pembuatan Kode Perantara)
Menghasilkan kode perantara yang lebih dekat dengan kode mesin tetapi masih independen dari mesin tertentu.
2. Code Optimization (Optimasi Kode)
Memperbaiki kode perantara untuk menjalankan program dengan lebih efisien dan cepat.
3. Code Generation (Pembuatan Kode)
Menghasilkan kode mesin yang dapat dieksekusi dari kode perantara.
4. Code Linking and Assembly (Penghubungan dan Perakitan Kode)
Menghubungkan kode objek yang dihasilkan dengan pustaka (library) dan menghasilkan program yang dapat dieksekusi.

Perbedaan Kompiler dan Interpreter

Kompiler

1. Proses Penerjemahan
Kompiler menganalisis seluruh kode sumber sekaligus dan mengubahnya menjadi kode mesin atau kode objek dalam satu tahap.
Hasilnya adalah file eksekusi yang dapat dijalankan langsung oleh komputer.
2. Kecepatan Eksekusi
Karena kode diterjemahkan seluruhnya sebelum dijalankan, program yang dikompilasi biasanya lebih cepat dalam eksekusi. Kode mesin yang dihasilkan telah dioptimalkan untuk kecepatan.
3. Deteksi Kesalahan
Kesalahan dalam kode (seperti kesalahan sintaks atau semantik) ditemukan selama tahap kompilasi sebelum program dijalankan. Program tidak akan dijalankan hingga semua kesalahan diperbaiki.
4. Contoh
Kompiler digunakan dalam banyak bahasa pemrograman seperti C, C++, dan Java (Java juga menggunakan interpreter untuk menjalankan bytecode).

Interpreter

1. Proses Penerjemahan
Interpreter membaca, menerjemahkan, dan mengeksekusi kode sumber baris per baris. Kode tidak diterjemahkan ke dalam bentuk eksekusi mandiri.
Interpreter bekerja secara langsung dengan kode sumber selama setiap eksekusi
2. Kecepatan Eksekusi
Eksekusi oleh interpreter umumnya lebih lambat dibandingkan dengan kompiler karena setiap baris kode diterjemahkan setiap kali dijalankan. Tidak ada proses optimasi sebelumnya.
3. Deteksi Kesalahan
Kesalahan hanya ditemukan ketika baris kode yang bermasalah dijalankan. Program mungkin berjalan sampai bagian yang bermasalah dan kemudian berhenti.
4. Contoh
Interpreter digunakan dalam bahasa seperti Python, Ruby, dan JavaScript. Bahasa ini sering digunakan untuk pengembangan cepat dan skrip.
