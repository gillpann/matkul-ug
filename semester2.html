- Basis Data -

Pengertian

Sistem File Basis Data adalah metode penyimpanan di mana data disimpan dalam satu atau lebih tabel dalam basis data. Sistem manajemen basis data (DBMS) menyediakan cara untuk mengelola, mengakses, dan memanipulasi data tersebut secara efisien.
Karakteristik Utama:
1. Penyimpanan Terpusat: Data disimpan secara terpusat dalam tabel-tabel yang saling terkait.
2. Kontrol Redundansi: DBMS mengelola dan mengurangi duplikasi data.
3. Independensi Data: Aplikasi tidak perlu mengetahui struktur data untuk mengakses data.
4. Keamanan Terpusat: DBMS menyediakan mekanisme keamanan untuk semua data.

Keuntungan

1. Integritas Data: DBMS memastikan integritas data melalui aturan dan batasan (constraints) yang diterapkan pada data.
2. Reduksi Redundansi: Data terpusat mengurangi duplikasi informasi yang sama di berbagai tempat.
3. Keamanan: DBMS menyediakan kontrol keamanan yang canggih, termasuk otorisasi akses dan enkripsi data.
4. Pemulihan Data: Kemampuan untuk memulihkan data dari keadaan gagal atau rusak melalui backup dan mekanisme pemulihan.
5. Kemudahan Akses dan Manipulasi: Query yang kuat memungkinkan pencarian dan manipulasi data dengan cepat dan efisien.

Kerugian

1. Biaya Implementasi: Biaya untuk membeli dan mengelola DBMS bisa tinggi, termasuk biaya lisensi dan perangkat keras.
2. Kompleksitas: Pengelolaan basis data memerlukan keterampilan teknis yang tinggi dan pemahaman mendalam tentang DBMS.
3. Kinerja: Untuk volume data yang sangat besar atau query yang kompleks, kinerja bisa menjadi masalah tanpa optimasi.
4. Ketergantungan pada DBMS: Kegagalan DBMS dapat mengakibatkan hilangnya akses ke data seluruhnya.
5. Pemeliharaan: Memerlukan pemeliharaan terus-menerus, termasuk backup, pembaruan, dan pengelolaan data.

- SQL (Structured Query Language) -

Definisi

SQL (Structured Query Language) adalah bahasa standar untuk mengelola dan memanipulasi basis data relasional. SQL digunakan untuk melakukan berbagai operasi pada data dalam basis data, termasuk pengambilan, penyisipan, pembaruan, dan penghapusan data, serta pengelolaan struktur basis data itu sendiri.

Karakteristik

1. Bahasa Deklaratif: SQL adalah bahasa deklaratif di mana pengguna menyatakan apa yang mereka inginkan, dan DBMS memutuskan bagaimana untuk mencapainya.
2. Standar: SQL memiliki standar yang ditetapkan oleh ANSI dan ISO, meskipun beberapa sistem basis data menambahkan ekstensi mereka sendiri.
3. Fungsi Luas: SQL mencakup perintah untuk definisi data, manipulasi data, kontrol data, dan query data.

Fungsi-Fungsi

1.Query Data: Mengambil data dari basis data menggunakan perintah SELECT.
2. Definisi Data: Mendefinisikan struktur basis data seperti tabel dan indeks menggunakan perintah CREATE.
3. Manipulasi Data: Menyisipkan (INSERT), memperbarui (UPDATE), dan menghapus (DELETE) data dalam basis data.
4. Kontrol Data: Mengatur hak akses pengguna ke data menggunakan perintah GRANT dan REVOKE.
